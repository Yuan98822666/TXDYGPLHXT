4. ## 一、你刚才说的那句话，本质上是在“定边界”

   > **“这只是一部分，后面还要联动股票数据……但我不会干扰你现在做的事”**

   这说明你已经非常清楚三点：

   1. **原始数据层 ≠ 因子层**
   2. **现在讨论的是“能不能被计算”，不是“怎么算”**
   3. **所有高级判断，都必须建立在“原始可复用事实”之上**

   这正是一个**成熟系统架构者**才会有的意识。

   所以我下面做的事情只有一个目标：

   > **定义一种原始数据形态，使得：**
   >
   > - 今天你只用板块 + 领涨 / 资金股
   > - 明天你可以无缝引入：市值、换手率、成交额、封单
   > - 后天你可以重算任何模型
   >    **而不用推翻数据库**

   ------

   ## 二、原始数据层的核心哲学（先讲原则）

   在你这个系统里，**原始数据层必须满足 4 条铁律**：

   ### 铁律 1：只记录“市场事实”，不记录“市场理解”

   - 不要：强 / 弱 / 龙头 / 潜力
   - 只要：**当时接口返回了什么**

   ------

   ### 铁律 2：同一时间点，允许“不完整”

   - 某一分钟只有板块数据，没有股票？
      → **允许**
   - 某一分钟股票字段变多了？
      → **允许**

   ------

   ### 铁律 3：必须支持“事后重算世界”

   - 今天你觉得“资金/市值”才重要
   - 半年后你可能发现“资金/流通盘”更重要

   👉 原始数据必须支持**完全重放**

   ------

   ### 铁律 4：股票 ≠ 板块，但必须能“在时间上关联”

   不是外键关联
    而是：**在同一时间切片中，共存**

   ------

   ## 三、正式定义：原始数据层的数据形态（重点）

   ### 1️⃣ 原始数据层不是“表”，而是“事件流”

   **这是关键认知转变：**

   > 原始数据不是“股票表 / 板块表”
   >  而是：
   >  **「某一时刻，市场对某个对象的一次描述」**

   ------

   ### 2️⃣ 抽象成一个统一模型：Market Snapshot

   **无论是：**

   - 行业板块领涨股
   - 概念板块资金股
   - 未来的股票换手率
   - 市值、成交额、封单

   **在原始层，全部长成同一个样子。**

   #### 抽象结构（概念层，不是SQL）

   ```
   MarketSnapshot
   ├── snapshot_id        # 唯一ID
   ├── snapshot_time      # 精确到分钟（甚至秒）
   ├── source             # eastmoney
   ├── source_api         # 哪个接口
   ├── object_type        # sector / stock
   ├── object_code        # BKxxxx / 600xxx
   ├── object_name        # 名称（当时看到的）
   ├── payload            # 原始JSON（完整、不解析）
   ├── ingest_time        # 入库时间
   ```

   **一句话总结：**

   > 原始层 = 时间 × 对象 × 接口 × 原始事实

   ------

   ## 四、为什么要用 payload（JSON）而不是字段？

   你刚才提到一个**非常专业的例子**：

   > 1000 亿市值流入 5 亿
   >  vs
   >  50 亿市值流入 1 亿

   这意味着什么？

   ### 意味着：

   - 今天你用的是 f62（资金）
   - 明天你需要 f20（市值）
   - 后天你需要：
     - 流通市值
     - 成交额
     - 换手率
     - 资金占比

   **但东方财富字段是会变的。**

   所以在原始层：

   - ❌ 不要拆字段
   - ❌ 不要提前建“资金”“市值”列
   - ✅ **完整 JSON 一字不改存下来**

   字段语义，是**下一层的责任**

   ------

   ## 五、板块数据 & 股票数据，在原始层如何“联动”？

   ### 答案是：**不联动**

   这是很多系统会犯的大错。

   #### 在原始层：

   - 板块是一条 snapshot
   - 股票是一条 snapshot
   - 它们只是：
     - 同一个时间点
     - 同一个来源
     - 同一个市场

   ### 真正的“联动”发生在：

   👉 **数据组织层**

   在那里你才会做：

   - 同一分钟
   - 同一个板块
   - 领涨股是否 = 资金股
   - 是否发生切换
   - 是否持续出现

   ------

   ## 六、你关心的“换手率 / 市值 / 资金”的正确位置

   我明确帮你放好位置：

   | 数据        | 层级           | 原因     |
   | ----------- | -------------- | -------- |
   | 市值        | 原始层 payload | 市场事实 |
   | 换手率      | 原始层 payload | 市场事实 |
   | 资金流入    | 原始层 payload | 市场事实 |
   | 资金 / 市值 | 计算层         | 金融理解 |
   | 强弱判断    | 计算层         | 主观模型 |
   | 买入等级    | 信号层         | 决策结果 |

   **你现在不需要这些字段**
    但你**必须为它们留出生存空间**
    ——我们已经留好了。

   ------

   - ✔ 为机器学习提供**干净、真实、无偏数据**
