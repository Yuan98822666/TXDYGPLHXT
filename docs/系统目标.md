# 系统目标：
​    1.架构方面：web系统 前后端分离 
​    2.使用目的：监控市场，识别出有涨停趋势的股票，在涨停之前识别出来，发出买入信号，由用户也就是我 来决定是否买入。
​    3.评价方面：成为天下第一的股票量化交易软件
​    4.功能方面：
​        4.1 计算板块强度，板块潜力，定位强力板块（允许多个板块很强）
​        4.2 计算个股强度，个股潜力，定位龙头个股（允许多个个股成为龙头）
​        4.3 根据板块轮动，预测下一个轮动板块（允许多个板块） 

# 需求澄清：
​    1.我要一个数据下载功能：下载市场 概念板块 领涨板块的数据 每一个板块都要包含领涨股 和 资金流入最多股信息。下载时间段为交易日的开市时间，下载频率[每分钟X次]和交易开关可在前端控制。
​    2.存储以上信息用于各种计算，所以计算频率也由前端控制[每分钟X次]，算法在进行编码时提供
​    3.计算结果前端图表展示
​    4.系统在计算得出买点后标记买入等级 一共5级 ，1级最不强烈，5级最强烈，记录每一个买入点，用于机器学习和回测。最终推荐的买入点越来越准确。
​    5.引入舆情监控，舆情分为财务造假、订单合同、产能突破、回购注销、扭亏为盈、业绩预期、减持、战争风险、政策等类型
