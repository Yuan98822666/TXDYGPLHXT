# 一、系统的本质定义（非常关键）

在正式画架构前，我们先统一一个“系统本质认知”，否则后面一定会走偏。

> **你的系统本质不是交易系统，而是：**
>
> **「A股主板 · 龙头形成过程的实时认知系统」**

关键词拆解：

- **不是自动交易** → 人在回路（Human-in-the-loop）
- **不是选股器** → 是“过程监控 + 信号提示”
- **不是静态因子模型** → 是“动态结构演化模型”
- **核心对象不是股票** → 是「**板块—资金—领涨—扩散—分歧—确认**」这一过程

所以系统架构必须服务于三件事：

1. **持续、稳定、高频地“看见变化”**
2. **把变化组织成“可计算的结构”**
3. **把结构变化转化为“可解释的信号”**

------

# 二、系统总体架构（宏观视角）

我先给你一个**逻辑分层架构**，不是技术选型，是“职责划分”。

```
┌────────────────────────────┐
│        前端展示层（Web）      │
│  图表 · 排名 · 信号 · 控制台   │
└────────────▲───────────────┘
             │
┌────────────┴───────────────┐
│        应用服务层（API）     │
│  调度 · 控制 · 查询 · 权限   │
└────────────▲───────────────┘
             │
┌────────────┴───────────────┐
│        计算与分析层          │
│  强度 · 潜力 · 轮动 · 信号   │
└────────────▲───────────────┘
             │
┌────────────┴───────────────┐
│        数据组织层            │
│  原始流 → 结构化 → 时间序列  │
└────────────▲───────────────┘
             │
┌────────────┴───────────────┐
│        数据采集层            │
│  东方财富接口 · 分页 · 校验  │
└────────────────────────────┘
```

**注意一句话：**

> 每一层都只做“自己该做的事”，绝不越权。

这是后面系统能否长期演进的生死线。

------

# 三、各层的“职责边界”（这是架构师的核心）

## 1️⃣ 数据采集层（Data Ingestion Layer）

### 角色定位

**“市场的忠实记录者”**

### 它只做三件事：

1. **按规则取数**
2. **不解释、不加工**
3. **原样落地**

### 架构原则

- 不关心“这是龙头还是垃圾”
- 不关心“有没有买点”
- 不关心“现在是不是强”

> 它的任务只有一句话：
>  **“市场说了什么，我一字不改地记下来。”**

### 重要设计决策（提前说清楚）

- 接口是**不可信的**（可能变字段、顺序、含义）
- 所以：
  - 原始返回必须完整保存
  - 解析逻辑必须版本化
  - 必须允许“历史重放”

------

## 2️⃣ 数据组织层（Data Organization Layer）

这是**整个系统最容易被忽视、但最核心的一层**。

### 角色定位

**“把杂乱市场信息，组织成‘可计算结构’”**

### 它解决的问题是：

- 分钟 A 和 分钟 B 之间 **发生了什么变化**
- 同一时间，不同板块之间 **谁更强**
- 同一板块内，领涨股是否 **发生切换**
- 资金股是否 **连续出现 / 消失**

### 重要原则

- 不做金融判断
- 只做“结构化”和“时间对齐”
- 让计算层可以**不用再关心原始脏数据**

你后面所有的：

- 板块强度
- 龙头识别
- 轮动预测

**全部建立在这一层是否干净、稳定、可回溯之上**

------

## 3️⃣ 计算与分析层（Analysis & Signal Layer）

### 角色定位

**“理解市场正在发生什么”**

这是你作为交易者最关心的一层，但**它不能污染下面的层**。

### 这一层才允许：

- 金融假设
- 市场经验
- 主观权重
- 策略演化

### 核心子模块（先点名，不展开）

- 板块强度模型（动态、相对）
- 板块潜力模型（趋势 + 加速度）
- 龙头形成模型（重复出现、扩散路径）
- 轮动模型（衰减 → 替代）
- 买入信号评分器（1–5级）

> **所有模型都必须满足一条铁律：**
>
> **同一份原始数据，不同模型可以反复重算**

------

## 4️⃣ 应用服务层（Application / API Layer）

### 角色定位

**“系统的大脑中枢”**

它不懂金融，也不抓数据，它负责：

- 前端控制采集 / 计算开关
- 调度频率（每分钟 X 次）
- 参数配置（模型权重、阈值）
- 权限、日志、状态管理

### 一个非常重要的设计思想

> **所有“可变的东西”，都应该在这一层**

这样你以后：

- 改频率
- 调算法
- 做A/B测试
- 回测旧版本

都不需要改底层。

------

## 5️⃣ 前端展示层（Web）

### 角色定位

**“把复杂结构，变成直觉”**

前端不是炫技，而是服务于你这类**盘中决策者**：

- 强板块一眼可见
- 龙头是否稳住一眼可见
- 资金是否持续一眼可见
- 信号强弱一眼可见

### 前端不是“看数据”

而是：

- 看**变化**
- 看**趋势**
- 看**异常**

------

# 四、为什么你说“不需要静态数据”是对的

你这个判断，**90%的散户系统是理解不了的**。

原因很简单：

> **在龙头形成阶段，“关系”比“属性”重要**

- 行业分类不重要
- 公司简介不重要
- 静态标签不重要

真正重要的是：

- 今天它是不是**被反复选中**
- 是不是**资金和情绪的交汇点**
- 是否处在**结构上升的节点**

所以：

- 静态表不是不需要
- 而是**不应该作为第一性存在**
