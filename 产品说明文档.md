## 一、系统概述

### 1.1 项目定位

天下第一股票量化系统是一个面向A股市场的全栈量化数据平台，提供：
- **实时数据采集**: 分钟级股票/板块快照
- **历史数据存储**: 日K、分钟级数据持久化
- **板块资金流向分析**: 概念/行业板块资金实时监控
- **特殊股票池跟踪**: 涨停、炸板、跌停股票监控
- **可视化展示**: 双折线图、统计面板、股票列表

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 双轨分类体系 | 概念板块(GN) + 行业板块(HY) + 申万三级行业 |
| 分钟级采集 | 交易时段内每分钟采集股票/板块快照 |
| 内存化调度 | TaskManager任务管理器，配置完全内存化 |
| 多维度分析 | 资金流向、涨跌幅、涨停状态、主力净流入 |
| 可视化大屏 | ECharts双折线图展示板块资金走向 |

## 二、核心功能模块

### 2.1 数据采集模块

#### 2.1.1 基础数据采集 (Base Collector)

**采集对象**:
- `base_stock`: 股票基础信息（代码、名称、交易所、板块类型、风险状态）
- `base_block`: 板块基础信息（代码、名称、类型GN/HY、成分股数量）
- `base_industry`: 申万三级行业分类（L1/L2/L3层级）
- `base_block_stock_lnk`: 板块-股票关联关系（约30万条）

**采集频率**: 每日开盘前/定期全量更新

#### 2.1.2 快照数据采集 (Raw Collector)

**股票快照** (`raw_min_stock`):
- 采集范围: `stock_imp=1` 的自选股
- 采集频率: 每分钟
- 数据字段: 价格(开/高/低/收)、成交量、涨跌幅、资金流向五档、涨停状态

**板块快照** (`raw_min_block`):
- 采集范围: 所有概念(GN) + 行业(HY)板块
- 采集频率: 每分钟
- 数据字段: 板块指数、涨跌幅、资金流向、涨跌家数、领涨股

#### 2.1.3 日K数据采集 (Day K Collector)

**采集对象**:
- `raw_day_stock`: 股票日K数据
- `raw_day_block`: 板块日K数据

**采集策略**: 支持 `append`(追加) 和 `replace`(替换) 两种模式

#### 2.1.4 特殊股票池采集 (Special Pool)

**监控对象**:
- `special_zt`: 涨停股票池
- `special_zrzt`: 昨日涨停今日表现
- `special_zb`: 炸板股票池
- `special_dt`: 跌停股票池
- `special_qs`: 强势股池

### 2.2 任务调度模块 (TaskManager)

#### 2.2.1 调度任务类型

| 任务名称 | 显示名称 | 调度方式 |
|----------|----------|----------|
| raw | 快照采集 | 区间执行(交易时段内) |
| special_pool | 特殊股票池采集 | 定点执行 |
| day_k | 日K采集 | 收盘后执行 |

#### 2.2.2 调度配置示例

```yaml
raw:
  enabled: true
  schedules:
    - name: "早盘连续竞价"
      type: interval
      start_time: "09:30:00"
      end_time: "11:30:00"
      interval_seconds: 60
    - name: "下午连续竞价"
      type: interval
      start_time: "13:00:00"
      end_time: "15:00:00"
      interval_seconds: 60
  scope:
    stock: true
    block: true
  threads:
    stock: 30
    block: 1
  delays:
    min_ms: 100
    max_ms: 200
```

#### 2.2.3 任务状态管理

- **IDLE**: 空闲状态
- **RUNNING**: 执行中
- **DISABLED**: 已禁用

### 2.3 板块资金流向模块 (BlockFlow)

#### 2.3.1 功能特性

**顶部统计栏**:
- 概念活跃数: 净流入>0的概念板块数/总数
- 行业活跃数: 净流入>0的行业板块数/总数
- Top5板块: 资金流入最多的5个板块（不分类型）

**双折线图**:
- 左图: 概念板块资金流向
- 右图: 行业板块资金流向
- 线条样式:
  - 仅资金流入Top10: 实线
  - 仅增速Top10: 虚线
  - 两者兼具: 粗实线(4px)
- 末端标签: 显示板块名称和最新资金流入

**股票列表**:
- 字段: 代码、名称、最新价、涨跌幅、主力净流入(亿)、所属板块
- 功能: 搜索、排序、分页、板块标签筛选

#### 2.3.2 API接口

| 接口 | 路径 | 功能 |
|------|------|------|
| 统计 | GET /api/block-flow/stats | 板块统计数据 |
| 时间序列 | GET /api/block-flow/timeseries | 分时数据 |
| 股票列表 | GET /api/block-flow/stocks | 板块成分股 |

## 三、数据架构详解

### 3.1 数据库表结构

#### 3.1.1 基础数据表 (base_*)

```
base_stock (股票基础表)
├── id (PK)
├── stock_code (UK) - 股票代码
├── stock_name - 股票名称
├── exchange - 交易所(0=深市, 1=沪市)
├── secid (UK) - 东方财富格式(如 0.000001)
├── stock_type - 板块类型(SH_ZB/SZ_ZB/KCB/CYB/BJS)
├── stock_risk - 风险状态(0=有风险, 1=正常)
├── stock_imp - 自选标记(0=未添加, 1=已添加)
├── sw_industry_l1/l2/l3 - 申万三级行业编码
├── skip_until - 跳过采集截止时间
└── pdate_time - 更新时间

base_block (板块基础表)
├── id (PK)
├── block_code (UK) - 板块代码(如 BK0428)
├── block_name - 板块名称
├── block_type - 板块类型(GN=概念, HY=行业)
├── block_stock_count - 成分股数量
└── update_time - 更新时间

base_industry (申万行业表)
├── id (PK)
├── level - 层级(1/2/3)
├── industry_code - 行业编码(如 L1_0, L2_12)
├── industry_name - 行业名称
├── em_bk_code - 东财BK码
├── parent_code - 父级编码
└── sort_order - 排序序号

base_block_stock_lnk (板块-股票关联表)
├── id (PK, BigInteger)
├── block_code - 板块代码
├── block_name - 板块名称(冗余)
├── stock_code - 股票代码
└── update_time - 更新时间
```

#### 3.1.2 快照数据表 (raw_min_*)

```
raw_min_stock (股票分钟快照)
├── id (PK, BigInteger)
├── stock_code - 股票代码
├── raw_no - 批次号(YYYYMMDDHHMMSS)
├── snapshot_time - 采集时间戳
├── trade_date - 交易日期
├── stock_zsj/kpj/zgj/zdj/spj/ztj/dtj - 价格相关
├── stock_cjl/cje - 成交量/额
├── stock_zdf/zf/zde/hsl/sjhsl - 涨跌幅/振幅/换手率
├── stock_syl/sjl/zsz/ltsz/ltg - 估值相关
├── stock_ztzt - 涨停状态(0=正常,1=涨停,2=炸板,3=跌停)
├── stock_zl/cd/dd/zd/xd_inflow - 资金流向五档(元)
├── stock_zl/cd/dd/zd/xd_zb - 资金流向占比(%)
└── created_at - 入库时间

raw_min_block (板块分钟快照)
├── id (PK, BigInteger)
├── block_code - 板块代码
├── block_name - 板块名称(冗余)
├── raw_no - 批次号
├── snapshot_time - 采集时间戳
├── trade_date - 交易日期
├── block_zs - 板块指数
├── block_ltg - 流通股
├── block_zdf/lb/hsl - 涨跌幅/量比/换手率
├── stock_cjls - 成交量(手)
├── block_up/pi/dw_stock - 涨跌家数
├── block_zl/cd/dd/zd/xd_inflow - 资金流向五档(元)
├── block_zl/cd/dd/zd/xd_zb - 资金流向占比(%)
├── leader_stock_code/name/zdf - 领涨股信息
├── money_stock_code/name - 资金流入最多股
└── created_at - 入库时间
```

## 六、项目优劣总结

### 6.1 项目优势

#### 6.1.1 架构设计优势

1. **双轨分类体系**
   - 概念板块(GN) + 行业板块(HY) + 申万三级行业并存
   - 满足不同分析场景需求
   - 板块-股票关联表设计合理(30万条数据)

2. **内存化任务调度**
   - TaskManager实现任务级启停控制
   - 配置完全内存化，支持动态修改
   - 轻量级调度，节省系统资源

3. **数据模型设计**
   - 基础表 + 快照表 + 日K表分层清晰
   - 冗余字段设计(如block_name)减少JOIN
   - 索引设计合理(高频查询字段均有索引)

4. **采集策略优化**
   - 多线程并发采集(股票30线程)
   - 延迟随机化(100-200ms)避免限流
   - 批次号设计支持数据追溯

#### 6.1.2 功能实现优势

1. **板块资金流向可视化**
   - 双折线图展示概念/行业资金流向
   - 线条样式区分(实线/虚线/粗实线)
   - 末端标签显示名称和金额
   - 固定交易时间轴(过滤午间休市)

2. **股票列表交互**
   - 板块标签可点击筛选
   - 支持搜索/排序/分页
   - 主力净流入单位自动转换(亿)

3. **特殊股票池监控**
   - 涨停/炸板/跌停/强势股多维度监控
   - 涨停状态自动计算(涨停/炸板/跌停)

### 6.2 项目不足

#### 6.2.1 架构层面

1. **数据一致性**
   - 分钟级数据采集已停采半月(最后数据2026-04-21)
   - 缺少数据完整性校验机制
   - 无数据补采功能

2. **错误处理**
   - API层错误处理不够完善
   - 缺少熔断/降级机制
   - 采集失败重试策略简单

3. **扩展性**
   - 股票代码硬编码较多
   - 采集器之间耦合度较高
   - 缺少插件化设计

#### 6.2.2 功能层面

1. **数据展示**
   - 板块资金流向缺少历史对比
   - 无个股资金流向趋势图
   - 缺少板块热度排行

2. **分析功能**
   - 无技术指标计算(MA/MACD等)
   - 缺少板块轮动分析
   - 无资金流向预测模型

3. **用户体验**
   - 前端缺少加载状态提示
   - 图表交互功能有限(无缩放/平移)
   - 缺少数据导出功能

#### 6.2.3 运维层面

1. **监控告警**
   - 无采集任务失败告警
   - 无数据延迟监控
   - 无系统资源监控

2. **日志管理**
   - 日志分散，无集中收集
   - 缺少结构化日志
   - 无日志分析工具

3. **部署运维**
   - 缺少Docker化部署
   - 无环境配置管理
   - 缺少数据库迁移脚本
